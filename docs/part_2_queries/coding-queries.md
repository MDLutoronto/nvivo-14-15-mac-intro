Coding Queries
--------------

1. Okay, now let’s move on. **Close any query results tabs that you have open**.

2. The next few queries I am going to show you are better to use when you have done some coding, as they can help you gain insight from all your work coding your content. For example, you could use these to find potential connections between codes, or connections between a code and a certain demographic.

3. Let’s start with Coding queries. You can use these to find all content coded at selected codes, a combination of codes, or a combination of codes and attributes. So for example, let’s say you want to see if there is content coded as both “Memorable quotes” and “Real estate development,” and you only want to see references that are from residents of Marshallberg – you can do this with a Coding query. **Go to the Explore menu, and from the Query drop\-down menu, select Coding.**

    <img src="{{ '/assets/images/NVivo15_Mac_intro_pt2_040.png' | relative_url }}" alt='Explore menu, with Queries and Coding highlighted' title='' width='1482' height='396' />

4. We can keep the defaults for now, which only finds references that are coded with all the codes listed. So, we just need to **click on the arrow** next to no selected codes to select the two codes we want: **“Memorable quotes”** **and “Real estate development.” Put checkmarks next to both of these codes**, and then **click on Select**.

    <img src="{{ '/assets/images/NVIVO_MAC_Coding%20Queries_4a.png' | relative_url }}" alt='Coding query window with no selected codes highlighted.' title='' width='698' height='205' /><img src="{{ '/assets/images/NVIVO_MAC_Coding%20Queries_4b.png' | relative_url }}" alt='Expanded Files folder tree with the Codes folder open and Memorable quotes and Real Estate development selected and outlined in red. Red box around Select.' title='' width='707' height='390' />

5. Then **click on Run Query** to see the results.

    <img src="{{ '/assets/images/NVIVO_MAC_Coding%20Queries_5.png' | relative_url }}" alt='Query window with Run Query highlighted.' title='' width='671' height='197' />

6. You are shown all the results, but you can also **click on the Summary tab just above the results** to see how many results you got. We can see there are a few results in interviews and survey responses.    

    <img src="{{ '/assets/images/NVIVO_MAC_Coding%20Queries_6.png' | relative_url }}" alt='In the Coding Query window, a red box outlines the Summary tab.' title='' width='717' height='307' />

7. But we can narrow this down further, to just quotes from residents of Marshallberg. To do that, we **can click on the plus sign at the top right** to add another parameter to our search.

    <img src="{{ '/assets/images/NVIVO_MAC_Coding%20Queries_7.png' | relative_url }}" alt='In the Coding Query Results Preview Window, a red box around the add parameter button.' title='' width='692' height='177' />

8. For this new row, **we** **use the drop\-down menu to select Any case where**.

    <img src="{{ '/assets/images/NVIVO_MAC_Coding%20Queries_8.png' | relative_url }}" alt='The code or case dropdown menu outlined in red with Any case where selected.' title='' width='740' height='223' />

9. Next **click on the arrow next to no selected attribute**.

    <img src="{{ '/assets/images/NVIVO_MAC_Coding%20Queries_9.png' | relative_url }}" alt='No selected attribute highlighted.' title='' width='673' height='202' />

10. Then **expand Person, select Township, and click Select Attribute.**

    <img src="{{ '/assets/images/NVIVO_MAC_Coding%20Queries_10.png' | relative_url }}" alt='Attribute list with Person highlighted and expanded and under it Township highlighted and selected. Select Attribute highlighted.' title='' width='435' height='439' />

11. **From the far blank right drop\-down, select “Marshallberg.”** So the query says “All” at the top left, meaning all of these conditions must be true. Then one line says that the content must be coded with both of the codes, Memorable quotes and Real estate development, and then the other line says AND the person who said this must be from Marshallberg.

    <img src="{{ '/assets/images/NVIVO_MAC_Coding%20Queries_11.png' | relative_url }}" alt='Red box around the open dropdown menu beside equals value, with Marshallberg selected.' title='' width='784' height='371' />

12. **Click on Run Query again**. This time you can see we get just one large memorable quote on real estate from Mary and James, who are from Marshallberg. (Click on the Reference tab to read it.)

    <img src="{{ '/assets/images/NVIVO_MAC_Coding%20Queries_12.png' | relative_url }}" alt='In the Coding Query window, in the results table, a red box around the one entry.' title='' width='720' height='301' />

13. We might not want to save the results, but we might want to save this query to re\-run again in the future as we continue to code. To do that, **click on Save Criteria**… at the top right.

    <img src="{{ '/assets/images/NVIVO_MAC_Coding%20Queries_13.png' | relative_url }}" alt='In the Coding Query Window, a red box around Save Criteria...' title='' width='757' height='223' />

14. Give it the name “Marshallberg Quotes on Real Estate” and click on **Save Criteria**. Then **close the query results tab using the View menu**.

    <img src="{{ '/assets/images/NVIVO_MAC_Coding%20Queries_14.png' | relative_url }}" alt='Save criteria window with the name field and Save Criteria highlighted. ' title='' width='503' height='269' />

15. This query is now saved in the **Query Criteria** section on the left, under **Queries**. You can double click on it at any time to re\-open the settings for the query, where you can then re\-run it.

    <img src="{{ '/assets/images/NVivo15_Mac_intro_pt2_053.png' | relative_url }}" alt='In the left menu, under Explore, a red box around Queries and Query Criteria. In the Queries list, a red box around Mashallberg Quotes on Real Estate. ' title='' width='2326' height='1322' />
